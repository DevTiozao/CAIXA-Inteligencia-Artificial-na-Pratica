# 🏗️ Roadmap de Carreira: Engenharia de Dados (Júnior)

Este documento contém o planejamento estratégico de 120 dias para a minha transição de carreira, focando na construção de infraestrutura e pipelines de dados.

---

## 🧩 VISÃO DO DIA A DIA
*O que faz um(a) Engenheiro(a) de Dados no cotidiano:*

* **Construção de Pipelines (ETL/ELT):** Criar os "canos" que levam dados de um lugar para outro de forma automática.
* **Modelagem de Dados:** Estruturar tabelas para que os analistas e cientistas encontrem informações rápido.
* **Monitoramento de Fluxos:** Garantir que os dados chegaram no destino sem erros ou perdas.
* **Qualidade de Dados:** Criar travas para que dados "sujos" ou errados não entrem no sistema.
* **Integração de APIs e Bancos:** Conectar diferentes sistemas (vendas, marketing, financeiro) em um repositório central.

## 📅 ROADMAP DE 120 DIAS (10h/semana)

### MÊS 3 - O MUNDO DO ETL

### MÊS 4 - PROJETO FINAL E MERCADO
* **Semana 13-14:** Construção do Projeto de Portfólio de ponta a ponta.

## 🚀 PROJETO DE PORTFÓLIO: Data Pipeline de Vendas
**Objetivo:** Criar um fluxo automatizado de dados de vendas simulados.

* **O que fazer:** Script Python que simula vendas (JSON), realiza limpeza (remover duplicados, padronizar datas) e carrega em PostgreSQL.
* **Entregáveis:** Repositório organizado, Diagrama de Arquitetura e Dashboard básico (Power BI/Excel).
* **Critérios de Aceitação:** Pipeline sem erros, dados formatados corretamente e script SQL de criação de tabelas.

---

**Dica:** Usar a bagagem em vendas para criar dados fictícios que reflitam status reais de pedidos!

---

## 💬 ROTEIRO DE ENTREVISTAS (Preparação)

1.  **Experiência Anterior:** Como a visão de vendas ajuda a entender o valor do dado para o negócio.
2.  **Processo ETL:** Explicar Extração, Transformação e Carga, focando na importância da Transformação.
3.  **SQL vs NoSQL:** Saber quando usar tabelas rígidas (consistência) vs documentos flexíveis (escala).
4.  **Versionamento:** A importância do Git para colaboração e rastreabilidade no pipeline.
5.  **Qualidade:** Como lidar com dados corrompidos usando filas de erro ou monitoramento.
